# IS
難易度は用意された段階ではなく「**演奏範囲（窓）**」＝ユーザーが担当する連続したキー範囲だ、という読み。曲ごとに選ぶ。

**原曲は壊さない。** `split()` が窓の内外へ `Score → Score` で可逆に割り、ユーザーは窓の中だけを弾く。窓の外は**消えるのではなくアプリが引き受け**、`merge()` が聴くときだけ合流させる。だから取り込み時に焼き付ける `monophonic` とは違い、練習の途中で窓を選び直せる。難易度調整の仕事は「元の譜面を壊さずに、あなたが触るキーを減らす」ことに尽きる。

- **窓を選ぶ画面**（`keywindow.py`・この機能の顔）: 鍵盤の絵に音の密度（`key_usage()`）を重ね、`fingers_needed()`（必要な指）と `note_share()`（あなたの音 / 全体）を**選ぶ前に**見せてドラッグさせる。`suggest_window()` は指 N 本で最も多くの音を弾ける連続窓を返し、同点なら狭い窓を採る（覚えるキーが少ないほど易しい）。
- **声部分離もメロディ検出もしない。** 音の密度を見せれば人間が見て決められる。きらきら星で「指 1 本」を押すと `C4〜A4` が選ばれるのは、旋律を当てているのではなく、密度と指の制約だけで結果的に一致するから。
- **窓は「オクターブ番号」ではなく連続したキー範囲**で定義する。MIDI のオクターブ番号は C4〜B4 で切れるので、番号で分けると「ドレミファソラシ**ド**」の最後の C5 がアプリに奪われる。最も単純な例がその定義では壊れる。
- **数えるのは音の数ではなくキーの数**。`KeyMapping.resolve()` はオクターブ移調で音域外を畳むため多対一で、C3 と C5 が同じキーに落ちる。2 音でも指は 1 本。難易度はマッピングに依存する。
- 窓は曲ごとに `config.key_windows` へ保存し、次からは黙って適用する。キー割り当てを変えて保存済みのキーが今の鍵盤に無くなったら未設定扱いに戻す（勝手に狭い範囲で練習させない）。
- `thin_chord` は難易度ではなく**任意の補助**（窓の中に和音があるが 1 本指で弾きたいとき）。窓が同時押しの上限を暗黙に決めるので、別軸として持つ必要が無い。
- **練習**（`practice.py`）: リズム型（自動で進む）／ ステップ型（押すと進む）・押下と離鍵の両方を判定・秒数シーク・参照五線譜・A-B ループ・速度倍率・メトロノーム（`audio`）・練習メモ（`practice_notes`）。

# PROCESS
- [done] 窓の分割ロジック（`difficulty.py`・`split` / `merge` / `apply`・`Score → Score` の可逆変換）
- [done] 窓を選ぶ画面（`keywindow.py`・曲ごとに保存・ツールバーから選び直せる）
- [done] 練習モード（リズム / ステップ・押下と離鍵の判定・シーク・参照五線譜・A-B ループ・速度倍率・メトロノーム・練習メモ）
- [done] 練習＋伴奏（窓の外をスピーカーで鳴らす。ゲームには一切送らない）
- [done] 不変条件をテストで固定（窓を広げると音は増えるだけで消えない / 予算を超えるキーは残らない）
- [todo] 窓の既定値（音符が最も多く集まる位置に置く。「何オクターブ」より「どのオクターブ」が効く）

# HISTORY
- **旧 IS「難易度 ＝ 同時に押すキーの数」**（`levels_for()` / `apply(max_keys)`・棄却）。窓が同時押しの上限を暗黙に決める（幅 8 鍵なら最大 8 本指、1 鍵なら必ず 1 本）ので、別軸として持つ必要が無かった。難易度は窓ただ一つ。速度と時間の進み方は最初から別軸として実装済みだった。
- **旧 IS「メロディ／伴奏に分離する」**（`split_for_accompaniment` / skyline・棄却）。分散和音（アルベルティ・バス）では何も減らず、保続音の下では伴奏の最高音が残り、内声メロディは落ちる。**ロジックは正しく、名前が嘘だった。** 袋小路だったのは「メロディ」という意味づけであって機能ではない。アプリが鳴らすのは `原曲 − あなたの譜面` ＝ ただの差集合で、何も推定しない、と立て直した。
