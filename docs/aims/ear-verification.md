# IS
音楽知識が無い人が採譜の正しさを確かめる手段は、譜面を読むことではなく**耳で聴いて元の曲と違うと感じること**だ、という読み。

**芽は既にある。** 音声プレビュー（キー送出なしでスピーカーから鳴らす）は今のところ「編集中の確認」という副次機能だが、本来は**自動採譜の直後に必ず通る検証ステップ**である。位置づけを変える。

フロー: 自動採譜 → 元音源と**交互に聴き比べ**（A/B）→ ズレを感じた箇所へジャンプ → **その小節だけトレースで直す** → 再度聴く。**五線譜を一度も読ませずに修正完了まで到達させる。**

- 流用できる機構: `audio.render_score_pcm` / `play_score`（`start_sec` / `end_sec` / `include_notes` / `metronome`）、練習モードの A-B ループと秒数シーク、`trace` の 2 点キャリブ、`difficulty.merge()`（一部だけを合流させて鳴らす）。
- **これが入るまで、採譜の出力は「下書き品質 → エディタで編集」という前提のまま**であり、ペルソナと矛盾する。譜面を読めない人は誤りを直せないどころか、**誤っていることにすら気づけない**。
- 入れば、採譜の precision の低さは「致命傷」から「直せる誤差」へ格下げできる。採譜エンジンの精度を上げるより安く、効く。
