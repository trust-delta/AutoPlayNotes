# IS
「自動演奏」で探す人は練習したい人と別のセグメントではなく、**あきらめた同一人物**である、という読み。「私は弾けないし…」という妥協の産物としてその語を打ち込んでいる。だから自動演奏は捨てるターゲットではなく**拾えるターゲット**であり、窓が空のときの縮退形として練習と同じ機構の上に載っている（→ [[playable-reduction]]）。

プロダクトの仕事は、**説教しないこと**。妥協を否定せず、ただ隣に道が続いていることを見せる。

- **補助演奏**（窓の外だけをゲームへ送る）がこの aim の実体。「キーを送るからグレーだ、だから作らない」は、全自動の人には何も要求せず、**6 鍵を自分で弾こうとした人にだけ「それは提供できません」と言う**ことになる。自動に頼らない方向を向く、その姿勢を肯定できなければ、主軸の転換は看板の掛け替えでしかない。
- **建前は捨てない。** キーは飛ぶ。グレーであることは UI にも README にも正確に書く。自動演奏は消さない・隠さない・**アイデンティティにはしない**。目当てで使うのは自己責任。
- ⚠️ **伴奏や補助演奏を実装しても、それ単体では誰も練習を始めない。** 自動演奏で来た人に足りないのは肩代わりではなく**導線**である。機能を足したことで導線を作ったつもりにならない。

- [todo] 自動演奏が終わったら「この曲、弾いてみますか？」を出す
