# IS
目的は**練習**であり、自動演奏はそこへ人を運ぶ入口＝同じ機構の縮退形（あなたの担当が空）だ、という読み。中心概念は「**演奏範囲（窓）**」＝あなたが担当する連続したキー範囲。難易度は用意された段階ではなく、曲ごとにユーザーが選ぶ窓そのもの。窓の外はアプリが引き受け、その引き受け方と直積を取ると、自動演奏から練習までが 1 本の連続した面の上に並ぶ。

| あなたの窓 | 窓の外をアプリが | モード | ゲームへのキー送出 |
|---|---|---|---|
| 空 | ゲームへ送出 | 自動演奏（**入口**） | あり |
| 1 窓〜 | ゲームへ送出 | 補助演奏 | あり |
| 空 | スピーカー | 単純再生（譜面確認） | なし |
| 1 窓〜 | スピーカー | 練習＋伴奏 | なし |
| 1 窓〜 | 鳴らさない | 練習 | なし |

- **窓で分ける必然**（実測）: あなたのキー集合とアプリのキー集合が全曲を通じて素になり、物理キーの取り合いが構造的に消える。同時押し数で分けると役割が時間をまたいで入れ替わり、アプリの keyup があなたの音を切る（OS のキー状態は 1 つしかない）。補助演奏が安全に成立する根拠。
- **窓を選ぶ画面**（`keywindow.py`・この機能の顔）: 鍵盤の絵に音の密度を重ね、`fingers_needed()`（必要な指）と `note_share()`（あなたの音 / 全体）を**選ぶ前に**見せてドラッグさせる。`suggest_window()` は指 N 本で最も多くの音を弾ける連続窓を返す。**声部分離もメロディ検出もしない** — 密度と指の制約だけで人間が決められる。窓は曲ごとに `config` へ保存。
- **譜面の分割**（`difficulty.py`）: `split()` が窓の内外へ `Score → Score` で可逆に割り、`merge()` が聴くときだけ合流させる。アプリが鳴らすのは「伴奏」ではなく `原曲 − あなたの譜面` ＝ ただの差集合で、何も推定しない。`thin_chord` は難易度ではなく任意の補助（窓の中の和音を 1 本指で弾きたいとき）。
- **素材の入口**（練習の前提。弾く曲が無ければ練習は始まらない）: 数字譜（`number_parser`）／ テキスト CDE（`text_parser`）／ MIDI（`midi_parser`, 任意依存 mido）／ MusicXML・MXL（`musicxml_parser`）／ 数字譜スクショ OCR（`ocr`, Windows 内蔵 OCR・依存ゼロ）／ PDF 各ページ画像化（`pdf`, 任意依存 pypdfium2）／ 楽譜画像を下敷きになぞるトレース入力（`trace`, 2 点キャリブ・**テールの主戦力**）。譜面が無い素材（画像・音源・映像）は、重依存を分離した子 aim `transcribe-addon`。
- **音長は演奏の質そのもの**: 対象楽器は「押している間だけ鳴り、押下エッジでしか発音しない」。`NoteEvent.durations`（音ごとの音長）・同キー衝突の後勝ち解決・`win_input.high_resolution_timer`（`timeBeginPeriod(1)`）で押下時間の誤差 ±0.8ms。⚠️ **この楽器挙動の出典はユーザーの知識であり、実機での再測定はまだ**。プリセットが前提する鍵盤数も未確認で、窓の既定値がそこに依存している。
- **練習**（`practice.py`）: リズム型（自動で進む）／ ステップ型（押すと進む）・押下と離鍵の両方を判定・A-B ループ・速度倍率・メトロノーム（`audio`）・練習メモ（`practice_notes`）。
- **周辺 / 配布**: `convert.py`（数字譜・キー文字譜・テキストの往復無損失）・`score_export.py`（MIDI / MusicXML 書き出し）・`staff`（五線譜プレビュー / 簡易エディタ）・`playlist`・`theme` / `config`・`gui`（customtkinter）。`build.py`（PyInstaller で単一 exe）。v0.1.0 リリース済み。

**残る穴**: 音楽知識ゼロの人が採譜の誤りに気づき、直す導線が無い。自動採譜は「下書き品質 → エディタで編集」を前提にするが、**譜面を読めない人は誤っていることにすら気づけない**。素材の入口を広げるほど、この穴は開く。

# HISTORY
- **旧 IS「最も広い入力 → 変換 → 2 系統の出力（自動演奏 / 練習）を対等に並べる」**（2026-07-09 棄却）。自動演奏で来る人は練習したい人と別のセグメントではなく、あきらめた同一人物である。対等な 2 出力として置くと**両者をつなぐ機構が設計に現れず**、実際 `Level` に速度と自動演奏フラグを混ぜる誤りを生んだ。自動演奏は出力ではなく、窓が空のときの縮退形。
- **旧 IS「難易度 ＝ 同時に押すキーの数」**（`levels_for()` / `apply(max_keys)`・棄却）。窓が同時押しの上限を暗黙に決める（幅 8 鍵なら最大 8 本指、1 鍵なら必ず 1 本）ので、別軸として持つ必要が無かった。難易度は窓ただ一つ。
- **旧 IS「メロディ／伴奏に分離する」**（`split_for_accompaniment` / skyline・棄却）。分散和音（アルベルティ・バス）では何も減らず、保続音の下では伴奏の最高音が残り、内声メロディは落ちる。**ロジックは正しく、名前が嘘だった。** 袋小路だったのは「メロディ」という意味づけであって機能ではない。差集合として立て直した。
- **旧前提「ゲーム内楽器はオンセットだけの世界だから音長は要らない」**（棄却）。`midi_parser` が `note_off` を捨て、`player` が `gate=40ms` を上限にして全曲スタッカートにしていた。3 層すべてが同じ誤った前提の上に建っていた。
